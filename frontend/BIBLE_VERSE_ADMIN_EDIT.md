## 🎯 기능 설명

### 관리자 권한 체크
- `isAdmin()` 함수로 관리자 여부 확인
- 관리자만 수정 버튼 표시

### 수정 버튼
- 각 구절 옆에 보라색 수정 아이콘 버튼
- 클릭 시 수정 모달 오픈

### 수정 모달
- 구절 정보 표시 (책, 장, 절)
- 텍스트 영역에서 내용 수정
- 빈 텍스트 방지
- 저장 중 로딩 상태 표시
- 에러 메시지 표시

### 낙관적 업데이트
- 저장 즉시 로컬 캐시 업데이트
- 화면에 즉시 반영
- 백그라운드에서 서버 동기화

## 📱 사용 시나리오

### 시나리오 1: 오타 수정
1. 관리자가 성경 읽기 화면에서 오타를 발견
2. 해당 구절의 "수정" 버튼 (보라색 연필 아이콘) 클릭
3. 텍스트 영역에서 내용 수정
4. "저장" 버튼 클릭
5. 즉시 화면에 반영

### 시나리오 2: 빈 구절 채우기
1. 관리자가 빈 구절 "(구절 내용 없음)" 발견
2. "수정" 버튼 클릭
3. 올바른 내용 입력
4. 저장하여 데이터베이스 업데이트

## ⚠️ 주의사항

1. **권한 체크**: 반드시 관리자 권한 확인 필요
2. **빈 텍스트 방지**: 빈 내용으로 수정 불가
3. **에러 처리**: 네트워크 오류, 권한 오류 등 적절한 에러 메시지 표시
4. **낙관적 업데이트**: 저장 후 로컬 상태 즉시 업데이트하여 UX 개선

## 🔌 백엔드 API 요구사항

### 엔드포인트
```
PUT /api/v1/bible/verses/{verse_id}
```

### 요청
```json
{
  "text": "수정된 구절 내용"
}
```

### 응답
```json
{
  "success": true,
  "data": {
    "id": 1,
    "book_id": 1,
    "book_name_ko": "창세기",
    "chapter": 1,
    "verse": 1,
    "text": "수정된 구절 내용"
  }
}
```

### 권한
- 관리자만 수정 가능
- 백엔드에서 JWT 토큰으로 관리자 권한 확인 필요

### 에러 응답
- `401 Unauthorized`: 인증 실패
- `403 Forbidden`: 관리자 권한 없음
- `404 Not Found`: 구절을 찾을 수 없음
- `400 Bad Request`: 빈 텍스트 또는 유효하지 않은 요청

## 🧪 테스트 방법

### 1. 관리자 로그인
```
username: admin
```

### 2. 성경 읽기 페이지 접속
```
/bible/study
```

### 3. 책과 장 선택
- 예: 창세기 1장

### 4. 수정 버튼 확인
- 각 구절 옆에 보라색 연필 아이콘 버튼이 보여야 함
- 일반 사용자는 버튼이 보이지 않음

### 5. 구절 수정
- 수정 버튼 클릭
- 모달에서 내용 수정
- 저장 버튼 클릭
- 즉시 화면에 반영 확인

### 6. 에러 케이스
- 빈 텍스트로 저장 시도 → 에러 메시지
- 네트워크 오류 시 → 에러 메시지
- 권한 없는 사용자 → 버튼 미표시

## 📝 추가 개선 사항 (선택)

1. **수정 이력 추적**: 누가 언제 수정했는지 기록
2. **일괄 수정**: 여러 구절 한 번에 수정
3. **원본 복원**: 수정 전 내용으로 되돌리기
4. **수정 승인 프로세스**: 수정 요청 → 승인 → 반영
5. **변경 사항 알림**: 수정된 구절을 읽은 사용자에게 알림
